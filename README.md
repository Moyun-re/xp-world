# XP大世界

> 一个纯前端、本地运行并可通过 GitHub Pages 访问的 XP / 设定条目管理工具。

当前版本：**v0.9**  
技术栈：**Vue 3 + Tailwind CSS + 单文件 HTML**  
项目入口：`index.html`

---

## 1. 项目简介

**XP大世界** 是一个用于管理、编辑、导入、导出设定条目的网页工具。

它不需要后端、不需要数据库，也不需要复杂安装。打开 `index.html` 即可使用，也可以部署为静态网页供其他用户访问。

主要特点：

- 纯前端单文件应用
- 支持本地 JSON 导入 / 导出
- 支持 A-Z 分类目录
- 支持搜索、折叠、批量选择、批量导出、批量删除
- 支持暗色模式
- 支持自动本地缓存，刷新页面后可恢复上次数据
- 使用自定义 Modal 弹窗，不依赖浏览器原生 `confirm`

## 3. 使用方法

### 3.1 直接打开

最简单的方式：

1. 打开项目文件夹
2. 双击根目录下的 `index.html`
3. 使用浏览器打开即可

推荐使用：

- Microsoft Edge
- Google Chrome
- Firefox

---

### 3.2 在线访问

如果项目已经部署到 GitHub Pages，直接打开项目的网页地址即可使用。

网页地址通常类似：

```text
https://你的用户名.github.io/仓库名/
```

---

### 3.3 导入数据

点击顶部按钮：

```text
导入
```

选择 `.json` 文件即可。

当前支持两种 JSON 格式：

#### 格式一：新版数组格式

```json
[
  {
    "title": "A1 - 示例世界",
    "content": "这里是正文内容"
  }
]
```

#### 格式二：旧版 SillyTavern / 世界书风格格式

```json
{
  "entries": {
    "0": {
      "comment": "A1 - 示例世界",
      "content": "这里是正文内容"
    }
  }
}
```

导入旧版格式时，程序会自动转换为当前内部格式。

---

### 3.4 编辑条目

1. 在左侧列表中点击一个条目
2. 在右侧编辑标题和正文
3. 修改会自动保存在浏览器本地缓存中

注意：

- 自动保存只保存在当前浏览器本地
- 如果更换电脑、更换浏览器或清理浏览器数据，缓存可能丢失
- 重要数据请及时使用「导出」备份

---

### 3.5 新建条目

点击顶部按钮：

```text
新建
```

可以选择分类基准，例如：

```text
A - 世界观-世界
B - 世界观-设定
C - 世界观-地点
```

选择分类后，系统会自动生成类似下面的标题：

```text
A1 - 自定义世界
```

也可以手动输入任意标题。

---

### 3.6 搜索条目

左侧搜索框支持搜索：

- 标题
- 正文内容

输入关键词后，列表会自动过滤匹配结果。

---

### 3.7 批量操作

左侧每个条目前有复选框。

选中多个条目后，左下方会出现悬浮操作栏，可执行：

- 导出选中条目
- 删除选中条目
- 清空选择

删除操作会弹出确认框，避免误删。

---

### 3.8 导出数据

点击顶部按钮：

```text
导出
```

会导出当前列表中的全部条目。

导出的文件名类似：

```text
XP大世界_全量_2026-06-20.json
```

建议每次大量修改后都导出一份备份。

---

### 3.9 清空数据

点击顶部按钮：

```text
清空
```

会清空当前浏览器中加载的全部条目。

这是危险操作，系统会弹出二次确认。

## 5. 数据安全说明

本项目没有后端，数据主要存在三个地方：

1. 当前页面内存
2. 浏览器 localStorage 本地缓存
3. 用户手动导出的 JSON 文件

请注意：

- 浏览器缓存不是永久保险箱
- 清理浏览器数据可能导致缓存丢失
- 更换浏览器后看不到原浏览器的数据
- 重要数据一定要定期导出 JSON 备份

推荐习惯：

```text
每次大量编辑后，点击「导出」保存一份 JSON。
```

## 6. 常见问题

### Q1：为什么刷新页面后数据还在？

因为当前版本已经加入本地缓存功能，会自动保存到浏览器 localStorage。

---

### Q2：为什么换浏览器后数据不见了？

因为 localStorage 是浏览器隔离的。Edge、Chrome、Firefox 的本地缓存互不共享。

解决方法：

1. 在原浏览器中导出 JSON
2. 在新浏览器中导入 JSON

---

### Q3：导入 JSON 后原来的数据会怎样？

当前逻辑是：**导入文件会替换当前列表中的数据**。

如果担心丢失旧数据，请先导出备份。

---

### Q4：这个项目需要 Node.js 吗？

不需要。

它是纯 HTML 前端项目，直接用浏览器打开即可。

---

### Q5：为什么页面样式没有加载出来？

本项目通过 CDN 加载 Vue 和 Tailwind：

```html
<script src="https://cdn.tailwindcss.com"></script>
<script src="https://unpkg.com/vue@3/dist/vue.global.js"></script>
```

如果网络无法访问这些地址，页面可能无法正常显示。

解决方案：

- 检查网络
- 使用可以访问 CDN 的环境
- 后续可改为本地化依赖版本
